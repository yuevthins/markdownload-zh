# MarkDownload 中文版 - 设计文档

> 创建于 2026-01-24 | Phase 1 产出

## 项目定位

面向 Obsidian 用户的中文优化 Markdown 网页剪藏 Chrome 扩展。

**核心差异化**：
- 更好的中文支持（微信公众号、知乎等）
- 符合用户 Vault 文件管理规范
- 简洁轻量，专注下载功能

## 目标用户

- 使用 Obsidian/Logseq 管理笔记
- 经常阅读中文网页内容
- 需要将文章保存为 Markdown 离线阅读

## MVP 功能清单

### 核心流程

```
点击扩展图标 → 提取页面内容 → 预览 Markdown → 编辑（可选）→ 下载 .md
```

### 功能列表

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 一键提取 | 使用 Readability.js 提取正文 | P0 |
| 实时预览 | Popup 中显示 Markdown 预览 | P0 |
| 快速编辑 | 下载前可修改标题 | P0 |
| 下载文件 | 保存为 `.md` 文件 | P0 |
| 中文文件名 | 自动清理特殊字符，限制 50 字符 | P0 |
| 基础模板 | 支持 Frontmatter 变量 | P1 |
| 微信公众号 | 特殊选择器提取 | P1 |

### 不在 MVP 范围

- 右键菜单
- 批量下载
- 复制到剪贴板
- 选中文本下载
- 配置页面
- 图片自动下载到附件目录

## 技术架构

### 开发框架：WXT

使用 [WXT](https://wxt.dev) 现代化扩展开发框架：

- **基于 Vite**：HMR 热重载，开发效率高
- **TypeScript**：开箱即用，类型安全
- **自动 Manifest**：根据文件结构自动生成
- **跨浏览器**：一套代码适配 Chrome/Firefox/Edge

```
┌─────────────────────────────────────────────────────────┐
│                   WXT Framework                          │
├─────────────┬─────────────┬─────────────────────────────┤
│   Popup     │  Content    │  Background                 │
│   (预览)    │  Script     │  Service Worker             │
│             │  (提取)     │                             │
├─────────────┴─────────────┴─────────────────────────────┤
│                      Utils                               │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐               │
│  │Readability│  │ Turndown │  │  中文    │               │
│  │  .js     │  │          │  │  处理器  │               │
│  └──────────┘  └──────────┘  └──────────┘               │
└─────────────────────────────────────────────────────────┘
```

## Vault 规范适配

### Frontmatter 模板

```yaml
---
title: "{{title}}"
id: {{id}}
created: {{date}}
updated: {{date}}
status: draft
category: resource
tags:
  - 收藏
source: {{url}}
---
```

### 模板变量

| 变量 | 说明 | 示例 |
|------|------|------|
| `{{title}}` | 页面标题 | 如何学习编程 |
| `{{url}}` | 页面 URL | https://example.com/article |
| `{{date}}` | 当前日期 | 2026-01-24 |
| `{{id}}` | 唯一 ID | 20260124-a3f9 |
| `{{content}}` | 正文内容 | Markdown 格式 |

### 文件名生成规则

| 规则 | 处理方式 |
|------|----------|
| 禁止字符 | 替换 `/\:*?"<>\|` 为 `-` |
| 长度限制 | 截断到 50 字符 |
| 空格处理 | 保留，不替换 |
| 中文 | 直接保留 |

### 图片处理

| 图片大小 | 处理方式 |
|----------|----------|
| ≤ 100KB | 转 Base64 内嵌 |
| > 100KB | 保留原始链接，提示手动处理 |

## 中文优化

### 微信公众号

- 选择器：`#js_content`
- 处理懒加载图片：`data-src` → `src`
- 移除：分享按钮、二维码、广告

### 知乎

- 选择器：`.RichContent-inner`
- 移除：卡片广告、推荐阅读

### 通用优化

- 中英文间自动加空格（可选）
- 统一标点符号
